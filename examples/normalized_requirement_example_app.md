# Normalized Requirement Example — App / PDA Demo

【建議 Demo 類型】
- App / PDA Demo
- 建議理由：此需求屬於現場收貨流程，操作步驟明確，使用情境適合手機 / PDA 任務式介面。

【模組名稱】
WMS 進貨作業

【使用者角色】
碼頭收貨人員

【使用情境】
收貨人員於碼頭使用 PDA 查詢採購單、輸入收貨資訊、處理異常，並完成 ERP 回寫。

【核心流程】
1. 搜尋採購單
2. 查看採購單明細
3. 點選明細並輸入收貨資料
4. 檢核後回寫 ERP，完成後返回搜尋首頁

【主要功能】
- 採購單搜尋
- 明細查看
- 收貨輸入與效期管理
- 超收檢核
- 拒收拍照
- 回寫 ERP

【頁面建議】
1. 搜尋頁
2. 採購單明細頁
3. 收貨輸入頁
4. 成功 / 異常提示 dialog

【查詢條件】
- 採購單號
- 供應商名稱
- 原料品號 / 品名

【主資料顯示欄位】
- 採購單號、供應商、預計到貨時間
- 品號、品名、採購量、已進量、未進量

【輸入欄位】
- 收料數量
- 效期
- 棧板 ID
- 拒收原因

【防呆 / 檢核規則】
- 超收檢查允收比例
- 多效期時綁定最短效期

【異常處理】
- 拒收
- 拍照
- 顯示異常說明

【我幫你先補上的合理預設】
- 搜尋支援關鍵字
- 標籤形式預設可選

【待你確認的項目】
- 是否要支援批號輸入
- 超收比例是否來自 ERP
