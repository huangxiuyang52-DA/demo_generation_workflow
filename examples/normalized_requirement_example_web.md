# Normalized Requirement Example — Web Demo

【建議 Demo 類型】
- Web Demo
- 建議理由：此需求屬於企業內部主資料維護，重點在查詢、列表、編輯與寬螢幕資料密度，適合 Web 後台管理畫面。

【模組名稱】
量測儀器維護

【使用者角色】
品保管理員 / 設備管理員

【使用情境】
使用者於桌面電腦維護儀器基本資料、校正資訊、狀態與備註，並透過查詢與表格快速檢視。

【核心流程】
1. 輸入查詢條件
2. 檢視表格列表
3. 開啟新增或編輯 modal
4. 儲存後更新列表資料

【主要功能】
- 查詢
- 新增
- 編輯
- 啟用 / 停用切換
- 分頁

【頁面建議】
1. 頁面標題區
2. 查詢條件區
3. 列表表格區
4. 編輯 modal

【查詢條件】
- 儀器編號
- 儀器名稱
- 儀器類別
- 啟用狀態

【主資料顯示欄位】
- 儀器編號、儀器名稱、類別、保管部門、校正週期、狀態
- 最後校正日、下次校正日、啟用狀態

【輸入欄位】
- 儀器編號
- 儀器名稱
- 類別
- 廠牌型號
- 校正週期
- 保管部門
- 備註

【防呆 / 檢核規則】
- 必填欄位檢查
- 日期格式檢查

【異常處理】
- 顯示驗證錯誤訊息
- 防止空白必要欄位儲存

【我幫你先補上的合理預設】
- 表格支援斑馬紋與 hover
- modal 使用 3~4 欄表單布局

【待你確認的項目】
- 是否需要附件 / 校正證書欄位
- 是否需要歷史異動紀錄
